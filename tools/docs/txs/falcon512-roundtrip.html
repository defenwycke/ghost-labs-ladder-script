How PQ Spending Works
1.
Funding transaction creates an output with rung_conditions containing a SIG block. The conditions store a 32-byte PUBKEY_COMMIT (SHA-256 hash of the full 897-byte Falcon public key) and the SCHEME byte 0x10 (FALCON_512).
2.
Spending transaction provides a ladder witness with the full Falcon-512 public key (897 bytes) and signature (657 bytes).
3.
Evaluator computes SHA256(pubkey) and compares against the committed hash — verifying the key matches what the output expected.
4.
Evaluator routes to rung::VerifyPQ() based on scheme byte. Falcon-512 signature is verified against the sighash using liboqs. Result: SATISFIED.

Falcon-512 is the most compact NIST-approved PQ signature scheme, making it the best candidate for Bitcoin transactions. At ~1.6 KB total witness it's roughly 14x larger than Schnorr, but still practical for on-chain use.
