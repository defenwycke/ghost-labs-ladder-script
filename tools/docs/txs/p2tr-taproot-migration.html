How P2TR Taproot Migration Works
1.
P2TR_LEGACY wraps the Taproot key-path spend as a typed block. The conditions commit to a 32-byte tweaked output key (x-only pubkey). Spending requires a single 64-byte BIP-340 Schnorr signature against the tweaked key.
2.
P2TR_SCRIPT_LEGACY wraps the Taproot script-path spend. The witness reveals a Merkle leaf and control block (internal key + Merkle path). The inner leaf must contain valid Ladder Script conditions that are deserialized and evaluated.
3.
The PQ_SAFE rung combines a CSV relative timelock (26000 blocks, ~6 months) with a FALCON-512 post-quantum signature. This path exists solely for emergency migration if classical cryptography is broken by quantum computers.
4.
This pattern demonstrates the migration path from legacy Bitcoin address types to native Ladder Script. Taproot key-path provides the best-case efficiency, script-path enables governed multisig, and the PQ rung provides forward-looking quantum safety — all within a single UTXO.
