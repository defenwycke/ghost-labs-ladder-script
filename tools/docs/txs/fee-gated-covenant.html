FEE-GATED COVENANT

Covenant with fee-rate hysteresis gating. The spend is only valid when network fee rates fall within a specified band, preventing execution during fee spikes. Combines HYSTERESIS_FEE with SIG and RECURSE_SAME.

SIG HYST_FEE AMT_LOCK SCHNORR · SCHEME
Rung Structure

Single rung with SIG + HYSTERESIS_FEE + RECURSE_SAME. The fee hysteresis band constrains when the covenant can be advanced.

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Fee-Gated Covenant Works
1.
The HYSTERESIS_FEE block defines a fee-rate band with high_sat_vb and low_sat_vb bounds. The evaluator checks the transaction's effective fee rate falls within this band.
2.
RECURSE_SAME enforces the carry-forward covenant — the output conditions are byte-identical, re-encumbering with the same fee gate.
3.
This pattern is useful for automated covenant chains that should only advance during low-fee periods, optimizing on-chain costs.
4.
If fees are outside the band, the rung is UNSATISFIED and the spend is rejected. The user must wait for fees to enter the acceptable range.
