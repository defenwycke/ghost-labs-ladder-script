SINGLE SIG (FALCON512)

Single post-quantum signature using FALCON-512 lattice-based cryptography. Compact PQ signatures (~690 bytes) with fast verification. Smaller than Dilithium3 but with different security assumptions.

SIG FALCON512 · SCHEME
Rung Structure

Single rung with one SIG block using SCHEME = FALCON512. More compact PQ signatures than Dilithium3.

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Single Sig (Falcon512) Works
1.
FALCON-512 produces compact PQ signatures (~690 bytes) compared to Dilithium3's ~3,293 bytes. Public keys are also smaller (~897 bytes).
2.
The condition encoding is identical: 32-byte PUBKEY_COMMIT with FALCON512 scheme in the coil. The evaluator selects the correct verification algorithm based on scheme.
3.
Total witness overhead is ~1,587 bytes (897B pubkey + 690B signature) — about half of Dilithium3 but still 16x larger than Schnorr.
4.
FALCON-512 is based on NTRU lattice problems and uses Gaussian sampling. It offers a good tradeoff between signature size and security level.
