2-of-3 MULTISIG VAULT

Multi-output vault pattern using the MULTISIG block type. A v4 RUNG_TX creates three outputs: a 2-of-3 threshold multisig for primary spending, a timelocked single-key recovery path (CSV), and change. The spend demonstrates satisfying the 2-of-3 threshold with two Schnorr signatures.

MULTISIG CSV SIG SCHNORR · SCHEME
Rung Structure

Two rungs define the spending paths: SPEND requires any 2-of-3 keyholders, RECOVER uses CSV + single-key recovery.

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How 2-Of-3 Multisig Vault Works
1.
Funding transaction creates three outputs. Vout 0 carries rung_conditions with a single MULTISIG block: NUMERIC threshold = 2, three PUBKEY_COMMITs, and SCHEME = SCHNORR.
2.
To spend, 2 of the 3 keyholders construct a transaction. The ladder witness contains paired PUBKEY + SIGNATURE fields for each signer.
3.
The evaluator reads the threshold (2) from the conditions, then validates each pubkey against the PUBKEY_COMMITs and each signature against the Ladder sighash. 2 valid signatures from distinct committed keys ≥ threshold. SATISFIED.
4.
Vout 1 provides a recovery path: after a CSV delay, key 1 alone can sweep. This is a separate UTXO, not an alternative rung — funds are split at funding time.
