DUAL SIG (COMPACT)

Two independent SIG rungs. Hot key for daily spending, cold key for recovery. Both rungs use SIG wire format &mdash; no block arrays needed. Common wallet pattern with minimal on-chain overhead.

SIG SCHNORR · SCHEME
Rung Structure

Two SIG rungs on separate outputs: HOT (daily spending key) and COLD (recovery key). Each uses SIG encoding.

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Dual Sig (Compact) Works
1.
Two outputs, each with a single SIG rung. The hot key output is for daily spending; the cold key output is for recovery.
2.
Both rungs are independently spendable — either key can authorize its respective output without the other. This provides key separation.
3.
SIG encoding on both rungs keeps the funding transaction minimal. Each output adds ~34 bytes of conditions overhead.
4.
This is the standard hot/cold wallet pattern: keep the hot key accessible for daily use, store the cold key securely for recovery.
