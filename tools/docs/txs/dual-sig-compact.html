DUAL SIG (COMPACT)

Two independent compact-encoded SIG rungs. Hot key for daily spending, cold key for recovery. Both rungs use COMPACT_SIG wire format &mdash; no block arrays needed. Common wallet pattern with minimal on-chain overhead.

How Dual Sig (Compact) Works
1.
Two outputs, each with a single COMPACT_SIG rung. The hot key output is for daily spending; the cold key output is for recovery.
2.
Both rungs are independently spendable — either key can authorize its respective output without the other. This provides key separation.
3.
COMPACT_SIG encoding on both rungs keeps the funding transaction minimal. Each output adds ~34 bytes of conditions overhead.
4.
This is the standard hot/cold wallet pattern: keep the hot key accessible for daily use, store the cold key securely for recovery.
