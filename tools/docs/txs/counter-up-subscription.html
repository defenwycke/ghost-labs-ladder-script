COUNTER-UP SUBSCRIPTION

Recurring subscription covenant: a counter increments on each payment (max N). The UTXO is amount-locked to a fixed payment size and auto-renews via RECURSE_MODIFIED. Three spend paths: PAY (increment counter), CANCEL (2-of-2 multisig), EXPIRE (counter reached target).

SIG CTR_UP AMT_LOCK REC_MOD SCHNORR · SCHEME
Rung Structure

Three rungs: PAY (SIG + CTR_UP + AMT_LOCK + REC_MOD), CANCEL (dual SIG cooperative exit), EXPIRE (CTR_UP at target + SIG for final withdrawal).

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Counter-Up Subscription Works
1.
The PAY rung enforces subscription logic: SIG authorizes, COUNTER_UP tracks payments (0/24), AMOUNT_LOCK constrains output to fixed range, RECURSE_MODIFIED increments the counter.
2.
Each PAY spend creates a carry-forward output with identical conditions except the counter's current field is incremented by delta via RECURSE_MODIFIED.
3.
AMOUNT_LOCK constrains the output to exactly min=max sats, ensuring each payment is a fixed amount. The fee comes from the surplus funded above the lock.
4.
After 24 payments, current = target = 24. COUNTER_UP becomes UNSATISFIED, locking PAY. The EXPIRE rung (requires counter at target) becomes spendable.
