CLTV_TIME CALENDAR LOCK

Calendar-based timelock using CLTV_TIME with Unix timestamp. Funds are locked until a specific calendar date rather than a block height. Useful for contracts tied to real-world dates.

CLTV_TIME H160_PRE SIG SCHNORR · SCHEME
DRY RUN — CLTV_TIME Jan 2027 unreachable — broadcast would fail but fund + sign shown
Rung Structure

Single rung with CLTV_TIME (Unix timestamp) + SIG. The funds unlock after a specific calendar date/time.

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Cltv_Time Calendar Lock Works
1.
CLTV_TIME checks that nLockTime (interpreted as Unix timestamp when ≥ 500,000,000) is at or past the committed timestamp.
2.
Unlike CLTV which uses block heights, CLTV_TIME allows locking to calendar dates. The evaluator compares nLockTime against the committed Unix timestamp.
3.
The SIG block requires authorization — reaching the unlock time alone isn't sufficient, the keyholder must also sign.
4.
This is a dry-run example: the timestamp is set in the future (Jan 2027), so the transaction is funded and signed but not broadcast.
