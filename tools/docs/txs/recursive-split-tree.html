RECURSIVE SPLIT (TREE)

Binary split covenant: a UTXO splits into 2 equal children on each spend. Each child inherits the same rung conditions with <code>max_splits</code> decremented. When splits are exhausted, the UTXO becomes freely spendable. Creates an on-chain binary tree of UTXOs.

Rung Structure

How Recursive Split (Tree) Works
1.
The funding TX encodes a RECURSE_SPLIT block with max_splits and min_sats. The evaluator requires the spending TX to produce exactly 2 outputs with the same conditions.
2.
Each child output inherits byte-identical conditions except max_splits is decremented by 1. When max_splits reaches 0, the UTXO is freely spendable (no covenant).
3.
The min_sats parameter prevents dust outputs — each child must carry at least this amount. The split divides the UTXO roughly in half.
4.
With max_splits=3, the tree expands: 1 → 2 → 4 → 8 UTXOs. This pattern enables mass distribution from a single funding transaction.
