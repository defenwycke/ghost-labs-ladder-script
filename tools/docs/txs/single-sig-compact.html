SINGLE SIG (COMPACT)

Simplest possible RUNG_TX: a single compact-encoded SIG rung. The wire format uses COMPACT_SIG encoding &mdash; just the pubkey commit and scheme byte, no block array. Minimal transaction size demonstrates the most efficient Ladder Script encoding.

SIG SCHNORR · SCHEME
Rung Structure

Single rung using COMPACT_SIG wire encoding. Instead of a block array, the conditions contain just the pubkey commit and scheme &mdash; the most compact possible encoding.

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Single Sig (Compact) Works
1.
COMPACT_SIG is a wire optimisation for single-SIG rungs. Instead of encoding a full block array (type code + field count + fields), the conditions contain just: pubkey_commit (32B) + scheme (1B).
2.
The evaluator transparently expands COMPACT_SIG into a SIG block evaluation. The signer provides the same witness: compressed pubkey (33B) + Schnorr signature (64B).
3.
This reduces the conditions overhead from ~38 bytes (full SIG block encoding) to ~34 bytes. The savings compound with multiple rungs or outputs.
4.
COMPACT_SIG is automatically used when a rung contains exactly one SIG block with SCHNORR or ECDSA scheme. PQ schemes use full block encoding.
