HTLC COMPACT SWAP

Compact HTLC atomic swap. The claim path combines hash preimage verification with signature authorization and CSV timelock. The refund path uses a CSV-locked SIG. Functionally identical to ATOMIC SWAP but using the compact HTLC block type.

How Htlc Compact Swap Works
1.
The HTLC block packs three checks into one: SHA-256 hash preimage verification, Schnorr signature validation, and CSV relative timelock — all in a single block type.
2.
To claim, the recipient provides the preimage and signature. The evaluator verifies SHA256(preimage) matches the hash, validates the signature, and checks CSV.
3.
This is more compact than using separate HASH_PREIMAGE + SIG + CSV blocks. The HTLC type code encodes the combined semantics in fewer condition bytes.
4.
The refund path activates after the CSV timeout. The sender signs with their own key to recover funds if the recipient doesn't claim.
