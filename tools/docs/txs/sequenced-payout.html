TRANSACTION EXAMPLE · PLC + SEQUENCING

SEQUENCED PAYOUT

Multi-stage payout with sequencer-enforced ordering. Each rung represents a payout stage. The SEQUENCER block tracks which stage is current and only allows spending the correct stage in order.

SEQUENCER A_ORACLE SIG AMT_LOCK REC_MOD SCHNORR · SCHEME
Rung Structure

Three sequenced rungs, each with SIG + SEQUENCER. The sequence counter ensures stages are spent in order: STAGE1 (current=0), STAGE2 (current=1), STAGE3 (current=2).

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Sequenced Payout Works
1.
The funding TX encodes three rungs with SEQUENCER blocks at different current values (0, 1, 2). Each rung also requires a SIG for authorization.
2.
The SEQUENCER evaluator checks current == expected_stage. Only the rung with the matching sequence number can be spent. This enforces strict ordering.
3.
After spending stage 0, only stage 1 becomes valid. After stage 1, only stage 2. This creates a deterministic payout schedule that cannot be reordered.
4.
Unlike timelocks which only enforce minimum wait times, the sequencer enforces strict ordering — you can't skip ahead or go backwards.
