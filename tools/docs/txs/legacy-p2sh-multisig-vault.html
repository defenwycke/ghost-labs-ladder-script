LEGACY P2SH MULTISIG VAULT

P2SH legacy wrapping inner Ladder Script conditions. The PREIMAGE carries serialized conditions that are deserialized and evaluated. Combines with P2WPKH hot-spend and CLTV cold sweep.

How Legacy P2SH Multisig Vault Works
1.
P2WPKH_LEGACY wraps SegWit v0 hash-based spending as a typed block. The conditions commit to a 20-byte witness program (HASH160 of compressed pubkey). Spending requires pubkey + signature in the witness.
2.
P2SH_LEGACY commits to the HASH160 of inner Ladder Script conditions. At spend time, the PREIMAGE reveals the serialized inner conditions. The evaluator hashes the preimage, checks it matches the commitment, then deserializes and evaluates the inner conditions.
3.
The inner conditions can be any valid Ladder Script block combination — multisig, timelocks, covenants. This gives P2SH_LEGACY the same composability as native Script-Hash while keeping the typed block safety guarantees.
4.
The COLD rung uses a CLTV absolute timelock at block 900000 combined with a single SIG. This acts as a cold-storage sweep path that only activates far in the future, providing a last-resort recovery mechanism.
