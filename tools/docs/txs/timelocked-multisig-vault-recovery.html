TIMELOCKED_MULTISIG VAULT RECOVERY

Vault with primary multisig path and timelocked recovery. The primary path requires 2-of-3 multisig. The recovery path activates after a CSV delay with a single recovery key. Combines threshold security with time-delayed fallback.

Rung Structure

Two rungs: SPEND (TIMELOCKED_MULTISIG with 2-of-3 threshold + CSV) and RECOVER (CSV-delayed SIG for single-key recovery).

Each row is a rung (spending path). Rungs are OR — satisfy any one rung to spend. Blocks within a rung are joined by + meaning AND — all must pass.

How Timelocked_Multisig Vault Recovery Works
1.
TIMELOCKED_MULTISIG combines threshold multisig with CSV timelock in a single block. The evaluator checks: M valid signatures from N committed keys AND nSequence satisfies the CSV delay.
2.
The primary path uses 2-of-3 with a short CSV. This ensures funds can't be spent instantly even with a quorum — there's always a small delay for monitoring.
3.
The recovery path uses a longer CSV delay with a single key. If the multisig keys are lost, the recovery key can sweep after waiting out the timeout.
4.
The tiered timelock design ensures: fast access with multisig quorum (short delay), slow access with single recovery key (long delay).
