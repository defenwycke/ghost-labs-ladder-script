Adaptor Sig Atomic Swap

Schnorr adaptor signature atomic swap round-trip. The funding transaction creates two spending paths: an ADAPTOR_SIG execute path (on-chain adapted signature reveals the adaptor secret for the counterparty to claim) and a timelocked CSV cancel path with dual SIG authorization. Compact SIG change output returns excess funds.

How Adaptor Sig Swaps Work
1.
Funding transaction creates outputs with rung_conditions. The EXECUTE rung contains an ADAPTOR_SIG block with two commitments: PUBKEY_COMMIT (SHA-256 of the signing key) and the adaptor_point T (a 32-byte x-only public key embedding the swap secret).
2.
To execute the swap, the claimer provides a Schnorr signature adapted with the secret scalar t corresponding to point T. The adapted signature s' = s + t is valid under the combined challenge H(R+T || P || m).
3.
Once the adapted signature appears on-chain, the counterparty can extract the adaptor secret: t = s' - s (difference between the adapted signature and the pre-signature). This reveals the secret needed to claim their side of the swap.
4.
The CANCEL rung provides a timeout path: after 288 blocks (~2 days), both parties can cooperatively recover funds using dual SIG authorization. This prevents indefinite fund locking if one party abandons the swap.
Adaptor Sigs vs HTLC Swaps

Secret Reveal Implicit (signature difference) Explicit (preimage in witness)

Witness Size ~110 bytes ~145 bytes
Quantum Safety Schnorr only Any scheme (PQ compatible)
Block Type ADAPTOR_SIG (0x0003) HTLC (0x0030) or decomposed

Adaptor signatures are the preferred swap mechanism when privacy matters. The on-chain transaction is indistinguishable from a regular Schnorr spend, revealing no hash or preimage. The trade-off is Schnorr-only (no PQ support) since adaptor signatures rely on the algebraic structure of elliptic curve signatures.
