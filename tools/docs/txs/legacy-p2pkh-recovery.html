How Legacy P2PKH + Recovery Works
1.
P2PKH_LEGACY wraps the classic Pay-to-Public-Key-Hash pattern as a typed Ladder Script block. The conditions commit to a 20-byte HASH160 of the spending public key.
2.
To spend via Rung 0, the witness provides the full compressed public key and a signature. The evaluator computes RIPEMD160(SHA256(pubkey)) and verifies it matches the committed hash, then validates the signature.
3.
If the primary key is lost, Rung 1 activates after 52560 blocks (~1 year). The CSV block enforces the relative timelock via nSequence, and the SIG block requires a valid signature from the recovery key.
4.
By wrapping P2PKH as a typed block, legacy address semantics are preserved without exposing arbitrary scriptSig data surfaces. The block type enforces exactly one spending pattern — no script injection possible.
