Block Reference

Documentation for each block type in the Ladder Script evaluation engine

Signature Family — 0x0001

Signature blocks verify cryptographic signatures against committed public keys. The foundation of UTXO authorization.

0x0001
SIG
Single Schnorr signature verification against a committed public key. Equivalent to OP_CHECKSIG in Tapscript.
0x0002
MULTISIG
M-of-N threshold signature verification. Conditions commit to N pubkeys and threshold M.
0x0003
ADAPTOR_SIG
Adaptor signature verification for atomic swaps and DLCs. Reveals a secret scalar upon completion.
0x0004
MUSIG_THRESHOLD
MuSig2/FROST aggregate threshold signature. On-chain footprint identical to single-sig.
0x0005
KEY_REF_SIG
Signature using a key commitment from a relay block. Cross-rung key sharing without duplicating pubkeys.

Timelock Family — 0x0100

Temporal constraints on spending — relative to confirmation (CSV) or absolute block height / timestamp (CLTV).

0x0101
CSV
Relative timelock by block count. UTXO cannot be spent until N blocks after confirmation.
0x0102
CSV_TIME
Relative timelock by time (512-second granularity).
0x0103
CLTV
Absolute timelock by block height. Equivalent to OP_CHECKLOCKTIMEVERIFY.
0x0104
CLTV_TIME
Absolute timelock by Unix timestamp using median-time-past.

Hash Family — 0x0200

Hash preimage verification.

0x0203
TAGGED_HASH
BIP-340 tagged hash preimage verification. Domain-separated hashing.
0x0204
HASH_GUARDED
Non-invertible SHA-256 preimage verification.

Covenant Family — 0x0300

Enforce constraints on how a UTXO can be spent by inspecting the spending transaction. No direct Tapscript equivalent.

0x0301
CTV
CheckTemplateVerify (BIP-119). Verifies spending transaction matches a committed template hash.
0x0302
VAULT_LOCK
Two-path vault with recovery key (immediate) and hot key (CSV-delayed).
0x0303
AMOUNT_LOCK
Output amount range check. Prevents dust and enforces spending caps.

Recursion Family — 0x0400

Self-referencing covenants — outputs that enforce spending back into the same or modified covenant structure.

0x0401
RECURSE_SAME
Enforces output recreates the identical covenant. Perpetual re-encumbrance.
0x0402
RECURSE_MODIFIED
Recursive covenant with allowed field mutations per iteration.
0x0403
RECURSE_UNTIL
Conditional recursion with termination condition. Bounded loops with exit criteria.
0x0404
RECURSE_COUNT
Count-limited recursion. Decrements counter each spend, terminates at zero.
0x0405
RECURSE_SPLIT
One input covenant fans out into multiple output covenants.
0x0406
RECURSE_DECAY
Each recursive spend reduces output value by a committed decay factor.

Anchor Family — 0x0500

Commit to external state references — channel IDs, pool IDs, reserve requirements, oracle attestations, tamper-evident seals.

0x0501
ANCHOR
Generic anchor point. 32-byte anchor hash identifying an off-chain structure.
0x0502
ANCHOR_CHANNEL
Payment channel anchor. Channel ID and sequence number.
0x0503
ANCHOR_POOL
Liquidity pool anchor. Binds UTXO to a specific pool structure.
0x0504
ANCHOR_RESERVE
Reserve proof anchor. Minimum reserve amount for proof of reserves.
0x0505
ANCHOR_SEAL
Tamper-evident seal. Single-use commitment for client-side validation.
0x0506
ANCHOR_ORACLE
Oracle attestation anchor. Spending requires oracle's signed event attestation.
0x0507
DATA_RETURN
Typed OP_RETURN replacement. Up to 32 bytes, max one per transaction.

PLC Family — 0x0600

Industrial automation primitives for Bitcoin script. Hysteresis, timers, latches, counters, comparators, sequencers, rate limiters.

0x0601
HYSTERESIS_FEE
Fee-rate hysteresis controller with deadband.
0x0602
HYSTERESIS_VALUE
Value-based hysteresis with high/low thresholds.
0x0611
TIMER_CONTINUOUS
Continuous on-delay timer. Spending allowed after preset duration elapses.
0x0612
TIMER_OFF_DELAY
Off-delay timer. Time-limited withdrawal windows.
0x0621
LATCH_SET
Set-dominant latch. Irreversible state transitions.
0x0622
LATCH_RESET
Reset-dominant latch. Inverse of LATCH_SET.
0x0631
COUNTER_DOWN
Down counter for N-spend limits.
0x0632
COUNTER_PRESET
Preset counter with configurable target. Reset on trigger.
0x0633
COUNTER_UP
Up counter for tracking spend count in stateful covenants.
0x0641
COMPARE
Numeric comparator with configurable operator (EQ, NE, LT, LE, GT, GE).
0x0651
SEQUENCER
Step sequencer. Ordered multi-phase spending policies.
0x0661
ONE_SHOT
Single-pulse trigger. Exactly one spend, then permanently disables.
0x0671
RATE_LIMIT
Maximum spends within a block-height window.
0x0681
COSIGN
Co-signing gate. Additional signature from a committed co-signer key.

Compound Family — 0x0700

Common multi-block patterns collapsed into single blocks, reducing wire overhead.

0x0701
TIMELOCKED_SIG
SIG + CSV in one block. Saves 8 bytes on wire.
0x0702
HTLC
Hash + timelock + signature. Replaces 3 blocks. Saves 16 bytes.
0x0703
HASH_SIG
Hash preimage + signature check. Saves 8 bytes.
0x0704
PTLC
Adaptor point + CSV + signature. For payment channels.
0x0705
CLTV_SIG
SIG + CLTV in one block. Saves 8 bytes.
0x0706
TIMELOCKED_MULTISIG
M-of-N + CSV for penalty branches and vault recovery.

Governance Family — 0x0800

Transaction-level constraints with no Tapscript equivalent. Inspect structure, weight, I/O counts, value ratios.

0x0801
EPOCH_GATE
Periodic spending windows using block height modular arithmetic.
0x0802
WEIGHT_LIMIT
Maximum transaction weight constraint.
0x0803
INPUT_COUNT
Min/max bounds on input count.
0x0804
OUTPUT_COUNT
Min/max bounds on output count.
0x0805
RELATIVE_VALUE
Anti-siphon ratio enforcement. Output/input value ratio check.
0x0806
ACCUMULATOR
Merkle set membership proof with sorted-pair proofs.
0x0807
OUTPUT_CHECK
Per-output value and script constraint.

Legacy Family — 0x0900

Wraps Bitcoin's original transaction types as typed Ladder Script blocks. Same spending semantics, all fields typed. Closes inscription and data-embedding vectors.

0x0901
P2PK_LEGACY
Pay-to-Public-Key. Delegates to EvalSigBlock.
0x0902
P2PKH_LEGACY
Pay-to-Public-Key-Hash. HASH160(pubkey) + signature check.
0x0903
P2SH_LEGACY
Pay-to-Script-Hash. Inner script must be valid Ladder Script conditions.
0x0904
P2WPKH_LEGACY
SegWit v0 key-hash spending.
0x0905
P2WSH_LEGACY
SegWit v0 script-hash with typed inner conditions.
0x0906
P2TR_LEGACY
Taproot key-path spend. Delegates to EvalSigBlock.
0x0907
P2TR_SCRIPT_LEGACY
Taproot script-path spend. Closes the primary inscription vector.
