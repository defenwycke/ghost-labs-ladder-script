BLOCK LIBRARY

Deep-dive documentation for each block type in the Ladder Script evaluation engine. Each page covers wire format, fields, evaluation logic, JSON output, and real-world use cases.

Signature Family — 0x0000

Signature blocks verify cryptographic signatures against committed public keys. The foundation of UTXO authorization — every spend ultimately requires at least one signature check.

0x0001
SIG
Single Schnorr signature verification against a committed public key. The most fundamental block type. Equivalent to OP_CHECKSIG in Tapscript.
Signature
0x0002
MULTISIG
M-of-N threshold signature verification. Conditions commit to N pubkeys and threshold M; witness provides M valid signatures.
Signature
0x0003
ADAPTOR_SIG
Adaptor signature verification for atomic swaps and DLCs. Verifies a signature that reveals a secret scalar upon completion.
Signature

Timelock Family — 0x0100

Timelock blocks enforce temporal constraints on spending — either relative to when the UTXO was confirmed (CSV) or absolute block height / timestamp (CLTV). Direct Tapscript equivalents exist for all four types.

0x0101
CSV
Relative timelock by block count. UTXO cannot be spent until N blocks after confirmation. Equivalent to OP_CHECKSEQUENCEVERIFY.
Timelock
0x0102
CSV_TIME
Relative timelock by time (512-second granularity). UTXO cannot be spent until N seconds after confirmation.
Timelock
0x0103
CLTV
Absolute timelock by block height. UTXO cannot be spent until block height reaches N. Equivalent to OP_CHECKLOCKTIMEVERIFY.
Timelock
0x0104
CLTV_TIME
Absolute timelock by Unix timestamp. UTXO cannot be spent until the median-time-past reaches the committed timestamp.
Timelock

Hash Family — 0x0200

Hash blocks verify knowledge of a preimage that produces a committed hash digest. Used for hash locks, atomic swaps, and commitment schemes. HASH_PREIMAGE has a direct Tapscript equivalent; HASH160 and TAGGED_HASH do not.

0x0201
HASH_PREIMAGE
SHA-256 hash preimage verification. Conditions commit to a 32-byte hash; witness provides the preimage. Equivalent to OP_SHA256 OP_EQUAL.
Hash
0x0202
HASH160_PREIMAGE
RIPEMD160(SHA256) hash preimage verification. 20-byte digest saves 12 bytes on wire vs HASH_PREIMAGE. Used for address-based hash locks.
Hash
0x0203
TAGGED_HASH
BIP-340 tagged hash preimage verification. SHA256(SHA256(tag) || SHA256(tag) || preimage). Domain-separated hashing for protocol-specific commitments.
Hash

Covenant Family — 0x0400

Covenant blocks enforce constraints on how a UTXO can be spent by inspecting the spending transaction itself. They enable pre-committed transaction templates, tiered vault custody, and output value restrictions — capabilities that have no direct Tapscript equivalent.

0x0401
CTV
CheckTemplateVerify (BIP-119). Verifies spending transaction matches a committed template hash covering version, locktime, outputs, and input index.
Covenant
0x0402
VAULT_LOCK
Two-path vault with recovery key (immediate) and hot key (CSV-delayed). Combines custody tiering and time-delayed authorization.
Covenant
0x0403
AMOUNT_LOCK
Output amount range check. Verifies min_sats ≤ output_amount ≤ max_sats. Prevents dust and enforces spending caps.
Covenant

Anchor Family — 0x0500

Anchor blocks commit to external state references — channel IDs, pool IDs, reserve requirements, oracle attestations, and tamper-evident seals. They bridge on-chain UTXOs to off-chain protocols and external data sources.

0x0501
ANCHOR
Generic anchor point. Commits to a 32-byte anchor hash identifying an off-chain structure. Base type for all anchor variants.
Anchor
0x0502
ANCHOR_CHANNEL
Payment channel anchor. Commits to a channel ID and sequence number. Enables channel state binding to on-chain UTXOs.
Anchor
0x0503
ANCHOR_POOL
Liquidity pool anchor. Commits to a pool ID binding the UTXO to a specific liquidity pool or coinpool structure.
Anchor
0x0504
ANCHOR_RESERVE
Reserve proof anchor. Commits to a minimum reserve amount, providing on-chain proof of reserves for off-chain balances.
Anchor
0x0505
ANCHOR_SEAL
Tamper-evident seal. Single-use commitment that can only be opened once. Used for client-side validation and single-use seals.
Anchor
0x0506
ANCHOR_ORACLE
Oracle attestation anchor. Commits to an oracle public key and event ID. Spending requires the oracle's signed attestation of the event outcome.
Anchor

Recursion Family — 0x0300

Recursion blocks create self-referencing covenants — outputs that enforce spending back into the same (or modified) covenant structure. They enable stateful contracts, progressive value decay, and bounded iteration without Turing-completeness.

0x0301
RECURSE_SAME
Enforces that an output recreates the identical covenant. The spent conditions must appear verbatim in the spending transaction's output.
Recursion
0x0302
RECURSE_MODIFIED
Recursive covenant with allowed field mutations. Specifies which fields may change between iterations while enforcing structural identity.
Recursion
0x0303
RECURSE_UNTIL
Conditional recursion — continues recursing until a termination condition is met. Enables bounded loops with explicit exit criteria.
Recursion
0x0304
RECURSE_COUNT
Count-limited recursion. Commits to a maximum iteration count; each spend decrements the counter. Terminates at zero.
Recursion
0x0305
RECURSE_SPLIT
Recursive split — one input covenant fans out into multiple output covenants. Enables tree-structured recursive expansion.
Recursion
0x0306
RECURSE_DECAY
Value-decaying recursion. Each recursive spend must reduce the output value by a committed decay factor, creating a diminishing covenant.
Recursion

PLC Family — 0x0600

PLC (Programmable Logic Controller) blocks bring industrial automation primitives to Bitcoin script. Hysteresis, timers, latches, counters, comparators, sequencers, and rate limiters — deterministic state machines for UTXO spending policy.

0x0601
HYSTERESIS_FEE
Fee-rate hysteresis controller. Allows spending only when fee rate crosses a threshold with deadband, preventing oscillation around boundary values.
PLC
0x0602
HYSTERESIS_VALUE
Value-based hysteresis controller. Triggers on UTXO value crossing high/low thresholds with deadband to prevent rapid state flipping.
PLC
0x0603
TIMER_CONTINUOUS
Continuous on-delay timer. Starts counting from UTXO confirmation; spending allowed only after the preset duration elapses continuously.
PLC
0x0604
TIMER_OFF_DELAY
Off-delay timer. Allows spending during a window after a trigger event, then locks. Used for time-limited withdrawal windows.
PLC
0x0605
LATCH_SET
Set-dominant latch. Once set by a trigger condition, remains set permanently. Creates irreversible state transitions in covenant logic.
PLC
0x0606
LATCH_RESET
Reset-dominant latch. Default state is set; a trigger condition resets it permanently. Inverse of LATCH_SET for complementary logic.
PLC
0x0607
COUNTER_DOWN
Down counter. Starts at a preset value and decrements each recursive spend. Triggers when count reaches zero. Used for N-spend limits.
PLC
0x0608
COUNTER_PRESET
Preset counter with configurable target. Counts up from zero; triggers when count reaches preset value. Reset on trigger.
PLC
0x0609
COUNTER_UP
Up counter. Increments each recursive spend with no upper bound. Used for tracking spend count in stateful covenants.
PLC
0x060A
COMPARE
Numeric comparator. Compares two values with configurable operator (EQ, NE, LT, LE, GT, GE). Used for conditional branching in covenant logic.
PLC
0x060B
SEQUENCER
Step sequencer. Advances through a committed sequence of states, one step per recursive spend. Enforces ordered multi-phase spending policies.
PLC
0x060C
ONE_SHOT
Single-pulse trigger. Allows exactly one spend, then permanently disables. The simplest state machine — a one-time authorization token.
PLC
0x060D
RATE_LIMIT
Spend rate limiter. Enforces a maximum number of spends within a block-height window. Combines counter logic with temporal constraints.
PLC
0x060E
COSIGN
Co-signing gate. Requires an additional signature from a committed co-signer key before the primary spending condition is evaluated.
PLC

Compound Family — 0x0700

Compound blocks collapse common multi-block patterns into single blocks, reducing wire overhead by eliminating duplicate block headers. Each compound block performs the evaluation of its constituent blocks in sequence.

0x0701
TIMELOCKED_SIG
Signature verification with relative timelock in a single block. Replaces SIG + CSV pattern. Saves 8 bytes on wire.
Compound
0x0702
HTLC
Hash Time Locked Contract — hash preimage verification + relative timelock + signature in one block. Replaces 3 blocks. Saves 16 bytes.
Compound
0x0703
HASH_SIG
Hash preimage verification combined with signature check. Replaces HASH_PREIMAGE + SIG. Saves 8 bytes on wire.
Compound

Governance Family — 0x0800

Governance blocks enforce transaction-level constraints that have no Tapscript equivalent. They inspect the spending transaction's structure, weight, input/output counts, and value ratios to enforce spending policies at the consensus level.

0x0801
EPOCH_GATE
Periodic spending windows — allows spending only during specific portions of each epoch. Uses block height modular arithmetic.
Governance
0x0802
WEIGHT_LIMIT
Maximum transaction weight constraint. Prevents the UTXO from being spent in oversized transactions.
Governance
0x0803
INPUT_COUNT
Bounds on number of inputs in the spending transaction. Min/max range enforcement for input consolidation control.
Governance
0x0804
OUTPUT_COUNT
Bounds on number of outputs in the spending transaction. Controls fan-out and prevents excessive output creation.
Governance
0x0805
RELATIVE_VALUE
Anti-siphon ratio enforcement. Ensures output value maintains a minimum ratio to input value. Uses integer ratio with overflow protection.
Governance
0x0806
ACCUMULATOR
Merkle set membership proof. Verifies that a value belongs to a committed set using sorted-pair Merkle proofs.
Governance
