HASH160_PREIMAGE

RIPEMD160(SHA-256) hash preimage verification. Uses the same double-hash as Bitcoin addresses for compact 20-byte hash commitments.

Wire Format Breakdown

Conditions side (in scriptPubKey):

0x0202 0 1 HASH160 · 20B = 26 bytes

Witness side (in input witness):

0x0202 0 1 PREIMAGE · 32B = 38 bytes
Total (conditions + witness) ~66 bytes

12 bytes smaller than HASH_PREIMAGE due to compact 20-byte hash commitment.

Evaluation Logic
1. PREIMAGE missing → ERROR
2. HASH160 missing → ERROR
3. Compute RIPEMD160(SHA256(PREIMAGE)). If result matches HASH160 → SATISFIED
4. Hash mismatch → UNSATISFIED

Use Cases
Bitcoin Address-Compatible Hashlocks
Hash commitments using the same HASH160 function as Bitcoin addresses. Allows reuse of existing address-generation tooling and familiar 20-byte hash format.
Compact Hash Commitments
When wire size matters, HASH160 saves 12 bytes per commitment compared to HASH256 (20 bytes vs 32 bytes) while maintaining strong preimage resistance.
Legacy Compatibility
Interoperability with systems that already use RIPEMD160(SHA256) for hash commitments, including existing Bitcoin script hash-lock patterns and P2PKH-derived protocols.
